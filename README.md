# Reyad Hasnain — Personal Website

A modern 2026 personal leadership website for **Reyad Hasnain** — ICT Policy Advocate, Telecom Strategist, Digital Economy Thought Leader, and Managing Director of Smart Lab Bangladesh.

Built with **Astro 6**, **React 19.2**, **Tailwind CSS v4 (Oxide)**, **Motion 12.38+** scroll-linked animations on the native `ScrollTimeline`, an OGL animated mesh-gradient hero shader, Astro Content Collections (MDX) for op-eds / TV / experience / policy / insights / speaking, an SMTP contact endpoint via Nodemailer + react-email templates, dynamic OG images via Satori, and full SEO 2026 (JSON-LD `Person` / `Article` / `VideoObject` / `Event` / `BreadcrumbList` / `FAQPage`, `llms.txt`, RSS, sitemap, AI-friendly `robots.txt`).

## Getting started

```bash
npm install
cp .env.example .env       # then fill in SMTP credentials
npm run dev                # http://localhost:4321
```

### Environment variables

See [.env.example](./.env.example).

| Var | Purpose |
|---|---|
| `PUBLIC_SITE_URL` | Canonical site URL (used in `<link rel="canonical">`, sitemap, OG meta) |
| `SMTP_HOST` `SMTP_PORT` `SMTP_SECURE` `SMTP_USER` `SMTP_PASS` | SMTP credentials for the contact form |
| `MAIL_TO` | Address that receives contact-form messages (`reyad@smart-lab.biz`) |
| `MAIL_FROM` | `From:` header on outgoing mail (must be authorised by your SMTP provider) |
| `RATE_LIMIT_PER_MIN` | Per-IP submission cap (default 5) |

Tested with Gmail App Passwords. Also works with Zoho Mail, Brevo SMTP, AWS SES SMTP, Mailgun SMTP.

## Performance & SEO

- **Astro 6 SSG** for every page except `/api/contact` and `/og.png` (which need runtime).
- **React Islands** with `client:visible` / `client:idle` / `client:load` so the only JS that ships is the JS needed for the visible interactivity.
- **Tailwind v4 Oxide** → 5× faster full builds, 100× faster incremental.
- **`@starting-style`** entrance animations are pure CSS — zero JS for above-the-fold reveals.
- **Motion v12.38+** runs scroll-linked animations on the browser's native `ScrollTimeline` for hardware acceleration.
- **OGL hero shader** (~10kb) loaded via `client:visible` — pauses when off-screen via `IntersectionObserver`, respects `prefers-reduced-motion`.
- **JSON-LD** on every page: `Person`, `WebSite`, `Article` (per op-ed), `VideoObject` (per TV appearance), `Event` (per speaking), `BreadcrumbList`, `FAQPage` (on About).
- **Dynamic OG images** at `/og.png?path=...&title=...` via Satori — fonts fetched once and cached in-process.
- **`llms.txt`** at root — Markdown bio for AI search engines (Anthropic, Vercel, Hugging Face standard).
- **AI-friendly `robots.txt`** with explicit allows for GPTBot, ClaudeBot, PerplexityBot, OAI-SearchBot, Applebot-Extended, Google-Extended.

## Accessibility

- WCAG 2.2 AA contrast targets (dark + light themes).
- `:focus-visible` rings, skip-to-content link.
- All animations respect `prefers-reduced-motion` (Lenis + OGL shader gate themselves off).
- Keyboard nav for the contact form.
- Semantic landmarks; ARIA only where HTML can't express.

## Deploy

The project ships with `@astrojs/node` standalone. To deploy:

```bash
npm run build
node ./dist/server/entry.mjs
```

Or use any Node host: Vercel, Railway, Render, Fly.io, Hetzner, your own VPS.
To swap to **Cloudflare Workers** when ready, replace the adapter in [astro.config.mjs](astro.config.mjs):

```js
import cloudflare from '@astrojs/cloudflare';
adapter: cloudflare({ mode: 'directory' }),
```

(Note: with Workers, the SMTP contact endpoint will need to be migrated to Workers' TCP socket API, or swapped to a fetch-based provider like Resend / Brevo HTTP.)
